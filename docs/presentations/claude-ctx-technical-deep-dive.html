System Topology: Plugin + CLI + TUI

Three layers cooperate: the Claude Code plugin ships assets, the Python CLI manages state, and the TUI visualizes and edits everything in real time.

Claude Code Plugin

Distributes agents, commands, modes, rules, and templates into the active Claude environment.

Python CLI

Source of truth for activation state, intelligence scoring, asset management, and exports.

TUI Dashboard

Live operator console for recommendations, activation, and navigation across assets.

Shared Roots

  • CLAUDE_PLUGIN_ROOT points to the plugin payload
  • ~/.claude stores active assets + state
  • .active-* files keep activation portable

Installation + Update Paths

Install via the plugin system, then use the CLI to initialize state, migrate legacy files, and keep assets aligned.

Plugin Install

  • make install or just install
  • Copies assets + completions + manpages
  • Sets up local plugin payload

CLI Setup

  • cortex setup migrate for legacy projects
  • cortex tui to launch the dashboard
  • Setup wizard fills missing templates

Activation + Recommendation Pipeline

Recommendations are derived from repo signals, scored against skill metadata, and persisted into activation state.

Pipeline Stages

  1. Detect: Git, file paths, commands, flags, profiles
  2. Enrich: Skill metadata, dependencies, tags
  3. Score: Weighted signals + confidence thresholds
  4. Recommend: Ranked candidates + rationale
  5. Activate: Apply profiles/agents/flags
  6. Persist: Update .active-* and metrics

Outputs

  • AI Assistant recommendations in TUI
  • Optional auto-activation for high confidence
  • Metrics stored for future weighting

Signal Sources + Context Features

Signals are pulled from both the repo and the operator session to keep recommendations relevant.

Repository Signals

  • Git status, diffs, commit messages
  • File paths, extensions, and directory patterns
  • Template presence and schema validation

Session Signals

  • Active agents, modes, profiles, flags
  • Recent commands and workflow runs
  • Skill rating history + usage frequency

Scoring + Confidence Controls

Recommendations are ranked using weighted heuristics with thresholds that guard against noisy activations.

Scoring Strategies

  • Weighted matches across tags, paths, and modes
  • Cooldowns to avoid repetitive prompts
  • Conflict suppression for mutually exclusive modes

Confidence Gates

  • Auto-activation only above threshold
  • Manual review for medium confidence
  • Ratings feed back into weighting

Intelligence Store + Learning

Cortex keeps long-term learning data in dedicated stores so recommendations improve over time.

Metrics

  • ~/.claude/.metrics/ tracks activations + usage
  • Skill rating prompts capture outcome quality
  • Session stats fuel recommendation tuning

Intelligence Cache

  • ~/.claude/intelligence/ stores patterns
  • Recency and frequency help weight future picks
  • Export pipelines reuse the same metadata

Watch Mode Loop

Watch mode runs the recommendation pipeline continuously with configurable thresholds and intervals.

Loop Steps

  1. Poll git + filesystem signals
  2. Run scoring engine
  3. Emit recommendations + telemetry
  4. Auto-activate when allowed

CLI Controls

  • cortex ai watch --threshold 0.8
  • cortex ai watch --interval 5.0
  • cortex ai watch --no-auto-activate

Skill Ratings + Feedback Loop

Ratings capture usefulness signals so the engine can weight trusted skills more heavily.

What Gets Rated

  • Prompted after skill execution or workflow runs
  • Usage frequency and success feedback
  • Recent outcomes influence ranking

Where It Shows Up

  • Skill list metadata + analytics
  • Recommendation weighting
  • Activation confidence tuning

Activation State + Guardrails

State is explicit and portable, with guardrails to keep templates aligned across upgrades.

State Files

  • FLAGS.md controls flag packs
  • .active-* files track modes, rules, profiles
  • CLAUDE.md is the main entry point

Guardrails

  • Template detection on launch
  • Setup wizard to initialize missing files
  • Migrate helpers for legacy state

TUI Intelligence Surfaces

Operators can launch recommendations, watch mode, and provider settings without leaving the dashboard.

Key Surfaces

  • AI Assistant view for recommendations
  • Command Palette for quick actions
  • Provider settings for multi-LLM consult

Operator Workflow

  • Review confidence + rationale
  • Activate with a single keystroke
  • Validate state in flags/modes views
